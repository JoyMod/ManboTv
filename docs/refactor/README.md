# MoonTV 重构计划: Next.js → Go + Gin

## 📋 项目理解总结

### 原架构 (Next.js Full-Stack)
```
┌─────────────────────────────────────────────────────────────┐
│                    前端层 (Next.js 14)                       │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────────────┐   │
│  │ React 页面  │ │ Tailwind UI │ │ PWA (离线缓存)      │   │
│  │ (App Router)│ │             │ │                     │   │
│  └──────┬──────┘ └─────────────┘ └─────────────────────┘   │
└─────────┼───────────────────────────────────────────────────┘
          │ API Routes (Next.js)
┌─────────┼───────────────────────────────────────────────────┐
│         ▼ 后端层 (Node.js Runtime)                           │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ API Routes: /api/*                                      ││
│  │ ├── search       - 多源聚合搜索 (核心瓶颈)              ││
│  │ ├── image-proxy  - 图片代理 (慢)                        ││
│  │ ├── proxy/m3u8   - 视频流代理                           ││
│  │ ├── favorites    - 收藏管理                             ││
│  │ ├── playrecords  - 播放记录                             ││
│  │ └── admin/*      - 管理后台                             ││
│  └─────────────────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────────────────┐│
│  │ 数据层: Redis / Upstash / Kvrocks / LocalStorage        ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

### 核心痛点
1. **搜索慢** - Node.js 单线程, 多源并发搜索阻塞
2. **图片加载慢** - Next.js 图片代理性能差, 无连接池
3. **并发差** - 高并发时响应延迟高
4. **内存高** - Node.js 运行时内存占用大 (100MB+)

## 🎯 新架构 (前后端分离)

```
┌─────────────────────────────────────────────────────────────────┐
│                      前端层 (保持原样)                           │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Next.js 14 (静态导出) → 纯静态文件                      │   │
│  │ • 剥离 API Routes, 改为调用 Go 后端                     │   │
│  │ • 保留 React + Tailwind + PWA                           │   │
│  │ • 构建输出: dist/ (静态文件)                            │   │
│  └─────────────────────────────────────────────────────────┘   │
└───────────────────────────┬─────────────────────────────────────┘
                            │ HTTP API
┌───────────────────────────▼─────────────────────────────────────┐
│                      API Gateway (Go + Gin)                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 核心特性:                                               │   │
│  │ • 连接池复用 (HTTP Keep-Alive)                          │   │
│  │ • Goroutine 并发 (每秒万级并发)                         │   │
│  │ • 内存 < 50MB                                           │   │
│  │ • 启动 < 1秒                                            │   │
│  └─────────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 路由层 (gin.RouterGroup)                                │   │
│  │ ├── /api/v1/search       - 聚合搜索 (Goroutine并发)     │   │
│  │ ├── /api/v1/image        - 图片代理 (连接池)            │   │
│  │ ├── /api/v1/proxy/*      - 视频流代理                   │   │
│  │ ├── /api/v1/favorites    - 收藏管理                     │   │
│  │ ├── /api/v1/playrecords  - 播放记录                     │   │
│  │ ├── /api/v1/live/*       - 直播源管理                   │   │
│  │ ├── /api/v1/admin/*      - 管理后台                     │   │
│  │ └── /api/v1/douban/*     - 豆瓣数据                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 服务层 (Service Layer)                                  │   │
│  │ • SearchService    - 多源并发搜索 + 超时控制            │   │
│  │ • ProxyService     - HTTP 代理 + 缓存                   │   │
│  │ • ImageService     - 图片代理 + 本地缓存                │   │
│  │ • StorageService   - 数据存储抽象层                     │   │
│  │ • LiveService      - 直播源管理                         │   │
│  │ • AdminService     - 管理后台                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 数据访问层 (Repository)                                 │   │
│  │ • Redis Client (go-redis) - 缓存 + 会话                 │   │
│  │ • SQLite (可选) - 本地配置存储                          │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

## 📊 性能对比预期

| 指标 | 原架构 (Node.js) | 新架构 (Go) | 提升 |
|-----|-----------------|------------|------|
| 并发处理能力 | ~1000 req/s | ~10000+ req/s | 10x |
| 内存占用 | 150-300MB | 20-50MB | 5x |
| 冷启动时间 | 3-5s | <1s | 5x |
| 图片加载 | 500ms+ | 100ms | 5x |
| 多源搜索 | 串行/慢 | Goroutine并行 | 3x |
